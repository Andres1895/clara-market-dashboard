## types
All CoinGecko API response shapes were defined in `src/types/coingecko.ts` before writing any API or component code. Defining the contract upfront ensures TypeScript catches shape mismatches at the data boundary rather than inside components, and gives the rest of the codebase a single source of truth for what the API returns.

## api
A dedicated CoinGecko API client was added in `src/api/coingecko.ts` as the single place all network calls originate — components and hooks never call `fetch` directly. The client implements a `RateLimitError` class for 429 responses so callers can distinguish rate limiting from other failures and surface a retry button rather than a generic error message.

- feat(api): add CoinGecko client with rate limit handling [e40e710]

## hooks
Three TanStack Query hooks wrap the API client: `useMarketData` for the top-coins list (auto-refreshes every 60 s), `useCoinDetail` for per-coin detail (enabled only when a coin id is provided), and `usePriceHistory` for 7-day OHLC data (also guarded by a null check). All hooks apply the project-wide 5 min stale / 10 min gc times and expose `isRateLimited` derived from the error type so UI layers can render actionable feedback without inspecting raw errors.
